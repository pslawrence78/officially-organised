# Tranche 10B.2: Event and Routine Entry Bridge v0.1

This tranche adds a lightweight bridge from **Add event** to the existing **Routines** creation surface so families can start from the familiar event flow and switch to the safer recurring model when a commitment repeats.

## What changed

- Added concise helper copy inside the Add Event form:
  - "Repeats regularly?"
  - "Use events for one-off plans. Use routines for clubs, lessons, weekly checks or repeat commitments."
- Added a secondary `Create routine` action in the Add Event flow.
- The action opens the existing `/routines` creation surface in create mode.
- Added optional transient prefill for safe fields only:
  - title
  - category
  - participants
  - responsible adults
  - place
  - car need status
  - start date
  - start time when available
  - derived duration when available
  - preparation task titles as routine defaults

## Why the models remain separate

Recurring routines still use the existing series model, recurrence validation, expansion, exceptions, and durable occurrence handling. One-off events still save as normal events. This tranche only improves discoverability between the two flows.

## Deliberately unchanged

- No Dexie schema changes
- No import/export changes
- No Supabase sync changes
- No recurrence model changes
- No event-to-routine conversion
- No recurrence controls added to the normal event form
- No primary navigation changes

## Validation

- Add Event now signposts routines for repeating commitments.
- Navigating to `Create routine` does not create an event or a routine.
- The routine form remains the source of truth for recurrence selection and save behaviour.

## Known limitation

The bridge uses transient route state only. If a user leaves the flow or reloads the page before saving, the optional prefill is intentionally lost rather than being persisted as a draft.
